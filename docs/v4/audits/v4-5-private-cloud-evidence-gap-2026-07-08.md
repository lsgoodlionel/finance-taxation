# V4-5 Private-Cloud Evidence Gap Audit

日期：2026-07-08  
范围：V4-5 私有云发布门禁中的安全、监控、对象存储、密钥管理证据链

## 结论摘要

- 当前仓库已有机器门禁：`load / backup-restore / connectors / ai-evals`。
- 当前仓库缺少四类私有云补充证据的结构定义、操作手册、样例与真实回填边界。
- 本次补齐目标是把缺口收束为可执行步骤和可回填字段，不改动现有工具主链。

## 缺口矩阵

### 安全

- 缺口：只有“V4-5 需要 MFA 或 SSO”这一规格要求，没有发布时必须提交的字段、导出位置和复核动作。
- 风险：无法核对高风险动作是否保留最终授权与审计证据。
- 本次补齐：新增安全 runbook 和 `security-access-review*.json` 模板。

### 监控

- 缺口：已有 `load.json` 性能门禁，但没有告警路由、告警演练和响应留痕要求。
- 风险：即使性能达标，也无法证明生产故障能被发现并升级。
- 本次补齐：新增监控 runbook 和 `monitoring-alert-drill*.json` 模板。

### 对象存储

- 缺口：规格要求对象存储，但现有门禁没有证明私有云桶、加密、保留策略和恢复验证。
- 风险：发布审批无法区分“本地落盘可用”与“私有云对象存储已达标”。
- 本次补齐：新增对象存储 runbook 和
  `object-storage-verification*.json` 模板。

### 密钥管理

- 缺口：规格要求密钥管理，但现有门禁没有记录密钥清单、轮换节奏、最近轮换和轮换后验证。
- 风险：生产凭据可能只停留在口头承诺，缺少可追溯轮换证据。
- 本次补齐：新增密钥管理 runbook 和 `key-management-rotation*.json`
  模板。

## 审计结果

1. V4-5 机器门禁无需变更，继续作为阈值校验主入口。
2. 私有云补充证据改为文档化门禁，由 manifest 汇总四类证据状态。
3. 样例文件必须显式标记 `sample: true`，真实发布前必须复制为非 sample
   文件并清空 `requiresBackfill`。

## 当前剩余动作

1. 在真实私有云环境中按四份 runbook 回填生产值。
2. 将样例文件复制为非 sample 文件并补上 `verifiedAt / verifiedBy`。
3. 把任何未完成项继续记录为新的审计条目，而不是保留模糊口头说明。
