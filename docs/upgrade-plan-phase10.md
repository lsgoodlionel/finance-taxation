## 二、升级方向建议（按优先级排序）

### Phase 10：质量加固与生产就绪（建议 2–3 周）

> **目标**：从"演示可用"推进到"生产可部署"

#### 10-A：安全加固（P0 关键）

| 任务 | 说明 |
|------|------|
| CSRF 防护 | 所有状态变更 POST/PUT/DELETE 加 CSRF token |
| CSP 头 | 配置 Content-Security-Policy，禁止 inline script |
| Rate Limiting | 登录/API 调用加频率限制（express-rate-limit） |
| 密码策略 | bcrypt 轮次提升、密码复杂度校验、登录失败锁定 |
| 输入验证统一 | 所有 API 端点引入 Zod schema 校验（当前约 30% 端点有校验） |
| 敏感数据脱敏 | 审计日志中的密码/token/银行账号脱敏 |
| 安全头 | Strict-Transport-Security / X-Frame-Options / Referrer-Policy |

#### 10-B：测试覆盖提升（P0 关键）

| 任务 | 说明 |
|------|------|
| E2E 测试框架 | Playwright 搭建，覆盖 5 条核心用户流 |
| 集成测试 | 后端 API 端点真实数据库集成测试（至少核心 CRUD） |
| 前端组件测试 | Vitest + Testing Library 覆盖关键交互组件 |
| 覆盖率门禁 | CI 中加入覆盖率检查，低于 60% 阻断 |

#### 10-C：性能基线（P1 重要）

| 任务 | 说明 |
|------|------|
| 数据库慢查询日志 | pg_stat_statements 或应用层 query timing |
| 前端 Bundle 分析 | vite-plugin-visualizer，确认当前 JS/CSS 体积 |
| 懒加载路由 | React.lazy + Suspense 按页面拆分 |
| API 响应缓存 | 科目主数据/税率规则等静态数据加 HTTP 缓存 |
| 首屏优化 | 关键 CSS 内联 + 字体预加载 |

#### 10-D：后端架构优化（P1 重要）

| 任务 | 说明 |
|------|------|
| `app.ts` 拆分 | 按模块域拆为独立路由注册文件，主入口 < 200 行 |
| 错误处理中间件 | 统一 error handler，消除各路由重复 try-catch |
| 请求日志 | 结构化 JSON 日志（pino/winston），含 requestId |
| 数据库连接池优化 | 连接池大小、超时、idle 配置 |

### Phase 11：多租户与权限精细化（建议 2 周）

> **目标**：支撑多企业 SaaS 运营

| 任务 | 说明 |
|------|------|
| 多租户数据隔离 | Row-Level Security（RLS）或 schema 隔离 |
| 字段级权限 | 部分角色只读特定列（如工资金额仅财务可见） |
| 操作审批流 | 可配置审批链（凭证/合同/工资三级审批） |
| 角色自定义 | 企业管理员可创建自定义角色 |
| 组织架构 | 支持多部门、多子公司 |
