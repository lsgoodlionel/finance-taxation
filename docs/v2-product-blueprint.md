# V2 产品蓝图

## 1. 产品目标

V2 的目标不是把当前原型继续扩充成“会生成单据的工具”，而是把它升级为一个面向科技型中小企业董事长、创始人、总经理的 `AI 财税工作台`。

目标能力：

- 董事长可以直接交办经营事件、审批事项、异常问题和管理目标
- 系统自动拆解为财务、税务、法务、行政、人事、业务协同任务
- 系统自动沉淀资料、生成底稿、推动执行、跟踪进度、预警风险
- 系统可以形成账、税、票、款、合同、研发、工资、报表之间的统一闭环
- 系统最终表现为“AI 财税负责人 + 多岗位协同平台”，而不是单纯记账软件

## 2. 目标用户

### 2.1 第一目标用户

- 科技型中小企业董事长 / 创始人
- 总经理 / COO
- 财务负责人 / 财务经理

### 2.2 第二目标用户

- 出纳
- 行政
- 人事
- 业务负责人
- 外部代账 / 税务顾问 / 审计顾问

## 3. 产品定位

### 3.1 要解决的问题

- 老板知道公司经营在发生什么，但不知道这些事情在财税上该如何落地
- 财务工作依赖单点人工经验，交办链路不可追踪
- 单据、合同、发票、回款、工资、研发、申报之间是割裂的
- 资料缺失通常在申报、审计、融资尽调时才暴露
- 研发费用、税收优惠、收入确认、回款风险无法被持续跟踪

### 3.2 V2 的定位

V2 定位为：

- `AI 财税办公室`
- `董事长财税驾驶舱`
- `财税任务执行系统`

而不是：

- 单纯票据识别系统
- 单纯记账系统
- 单纯税表生成器

## 4. 参考标杆

V2 应当组合借鉴成熟开源项目的长处，而不是照抄单一产品：

- `ERPNext`
  - 借鉴：业务、财务、采购、库存、项目、HR 一体化主线
  - 借鉴点：统一业务对象、统一状态流、统一报表
- `Akaunting`
  - 借鉴：中小企业财务主应用 + 模块扩展
  - 借鉴点：模块化、轻部署、面向中小企业的使用路径
- `LedgerSMB`
  - 借鉴：严谨的双分录、总账和报表基础
  - 借鉴点：账务内核、过账控制、凭证与报表关系
- `Dolibarr`
  - 借鉴：CRM、订单、供应商、单据协同
  - 借鉴点：从经营事项直接推动财务动作
- `TaxHacker`
  - 借鉴：上传即结构化、自动分类、AI 辅助判断
  - 借鉴点：低门槛输入体验与智能抽取

## 5. 核心设计原则

- 所有 AI 结论必须可解释、可回放、可追溯
- 所有高风险结论必须要求人工确认
- 所有经营行为必须落成任务、时限、责任人、资料清单
- 所有业务对象必须能够关联合同、发票、回款、分录、税务影响
- 所有老板视角问题必须有统一口径的数据来源

## 6. V2 产品结构

### 6.1 董事长驾驶舱

核心视图：

- 现金余额与未来 90 天资金预测
- 本月收入、成本、毛利、费用、利润
- 应收回款进度与逾期风险
- 应付付款计划与资金占用
- 本月应申报、应缴税、已完成、异常项
- 高风险事项清单
- 待董事长审批 / 决策事项
- AI 今日工作摘要
- 财务报表中心快捷入口
- 研发辅助账快捷入口
- 风险勾稽中心快捷入口

主要动作：

- 直接用自然语言交办
- 批准 / 驳回 / 延期任务
- 查看异常成因与处理建议
- 查看月度、季度、年度财税健康度

### 6.2 AI 财税秘书

输入来源：

- 文本行为录入
- 合同
- 发票
- 银行回单
- 工资表
- 会议纪要
- 邮件摘要
- 上传附件

输出结果：

- 经营事项识别
- 会计建议
- 税务建议
- 风险提示
- 任务拆解
- 所需资料清单
- 生成单据建议
- 生成报表影响

### 6.3 经营事项总线

统一管理所有经营事件：

- 销售

- 借款融资
- 政府补助
- 研发项目
- 税务申报

每个经营事项都要带：

- 来源
- 金额
- 时间
- 部门
- 责任人
- 当前状态
- 关联合同
- 关联发票
- 关联付款 / 回款
- 关联任务
- 关联凭证 / 分录
- 关联风险

### 6.5 税务运营中心

### 6.7 风险与勾稽中心

### 6.8 归档与审计中心

## 14. V2 边界

V2 的目标是做成“可替代大量日常财税执行工作”的系统，但不应在产品承诺上越界：

- 不承诺完全替代法定财务负责人
- 不承诺自动完成官方申报系统最终提交
- 不承诺替代法律、税务、审计专业判断

V2 应承诺的是：

- 大幅减少人工整理、催办、归档、汇总、追踪、复核的工作量
- 提供统一、透明、可追溯的财税执行机制
- 让董事长和财务负责人随时知道公司财税工作的真实状态
