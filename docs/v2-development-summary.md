# V2.0 开发总结

日期：2026-06-08（最后更新）

用途：
- 作为新窗口接续文本
- 作为 V3.0 页面与交互重构的业务基线
- 作为当前代码库的 V2 功能闭环总结

---

## 1. V2.0 的总体结果

V2 已完成从“高保真原型 + 零散工具页”到“可执行财税工作台”的第一轮建设，当前主链已经明确：

`AI 财税助手 / 经营事项总线 -> 任务中心 -> 单据中心 -> 凭证中心 -> 总账中心 -> 财务报表 / 税务中心 -> 风险勾稽 / 审计日志 -> PDF 导出`

V2 的重点不是页面美观，而是把业务链跑通。到当前为止，这条链已经具备可用的前后端对象、页面承接、导出入口和第一轮风险审计闭环。

## 2. V2 已完成的核心能力

### 2.1 事项主线

- `AI 财税助手` 支持：
  - 多轮对话
  - OCR/PDF 识别
  - 识别后生成经营事项建议
  - 自动生成事项并触发分析
- `经营事项总线` 支持：
  - 新建事项
  - AI 分析
  - 状态推进
  - 风险检查
  - 关联任务/单据/凭证/税务查看

### 2.2 执行与资料

- `任务中心` 已形成任务树、状态流、逾期过滤
- `单据中心` 已支持：
  - 单据列表/详情
  - 报销类正式模板视图
  - 附件上传
  - 归档
  - 跳转到任务/凭证/税务

### 2.3 账务与报表

- `凭证中心` 已支持：
  - 模板制单
  - 校验
  - 审核
  - 过账
  - 过账记录查看
- `总账中心` 已支持：
  - 分录
  - 科目余额
  - 日记账
  - 期间锁账
- `财务报表` 已支持：
  - 资产负债表
  - 利润表
  - 现金流量表
  - 快照
  - 差异分析
  - 老板摘要
  - 打印

### 2.4 税务、研发、风控

- `税务中心` 已支持：
  - 纳税人口径
  - 税率与期间规则
  - 税务事项
  - 增值税底稿
  - 企业所得税准备
  - 个税资料
  - 印花税与附加税摘要
  - 申报批次、复核、留档
- `研发辅助账` 已支持：
  - 项目
  - 成本
  - 工时
  - 加计扣除资料包
- `风险勾稽` 已支持：
  - 规则检查
  - 风险评分
  - 关闭与复盘记录
- `审计日志` 已具备对象级日志查看基础能力

### 2.5 导出与归档

- `PDF 导出中心` 已统一承接：
  - 报表快照
  - 税务底稿
  - 企业所得税准备稿
  - 月结/审计/稽核资料包
  - 工资资料
  - 凭证
  - 单据正式模板
  - 风险复盘
  - 研发资料包

## 4. V2 的当前业务基线

当前应把 V2 视为：

- 后端业务对象和页面功能已经能跑通主链
- 复杂对象之间已存在第一轮 drilldown 和页面穿透
- 合同、工资、导出、风险这些原本孤立或薄弱页面，已经接回主线

也就是说，V3 不需要再“从无到有补功能”，而应该：

1. 复用 V2 已有业务对象和 API
2. 重构高频页结构
3. 统一设计系统、状态、空态、流程表达
4. 让页面从“工程可用”提升到“产品可用”

---

## 5. 为什么进入 V3

V2 的问题已经从“功能缺失”转变为“页面组织方式落后于功能复杂度”。

当前的主要问题是：

- 核心页文件仍然过大
- 列表/详情/操作/引导常混写在单页
- URL 状态和页面壳不统一
- 设计系统和共享 UI 仍不完整
- 高频路径缺少统一的“入口 -> 过程 -> 结果”表达

因此 V3 的核心目标不是改业务规则，而是改页面结构和交互体系。
