# 页面功能与业务逻辑梳理

日期：2026-05-22

范围：左侧导航从 `董事长驾驶舱` 到 `PDF 导出` 的全部页面，梳理页面职责、相互之间的业务逻辑关系，并检查是否存在问题或缺失。

---

## 1. 页面与职责

### 1.1 概览

- `董事长驾驶舱`

  - 职责：汇总经营、利润费用、风险待办、AI 工作摘要

### 1.2 经营管理

- `经营事项总线`
  - 页面：`apps/web/src/pages/EventsPage.tsx`
  - 职责：创建事项、AI 分析、状态推进、触发风险检查
  - 定位：核心主入口之一

- `合同管理`
  - 页面：`apps/web/src/pages/ContractsPage.tsx`
  - 职责：新建合同、查看合同详情、履约/终止
  - 定位：业务主数据页

- `工资管理`
  - 页面：`apps/web/src/pages/PayrollPage.tsx`
  - 职责：员工管理、工资计算、工资确认、工资参数
  - 定位：人力与薪酬业务页

- `任务中心`
  - 页面：`apps/web/src/pages/TasksPage.tsx`
  - 职责：查看任务树、变更状态、催办、逾期过滤
  - 定位：执行流转页

### 1.3 财务运营

- `单据中心`
  - 页面：`apps/web/src/pages/DocumentsPage.tsx`
  - 职责：查看单据、正式模板展示、上传附件、归档、跳转到任务/税务/凭证
  - 定位：原始资料与业务单据沉淀页

- `凭证中心`
  - 页面：`apps/web/src/pages/VouchersPage.tsx`
  - 职责：模板制单、校验、审核、过账、查看过账记录
  - 定位：记账前置核心页

- `总账中心`
  - 页面：`apps/web/src/pages/LedgerPage.tsx`
  - 职责：总账分录、科目余额、日记账、期间锁账
  - 定位：账务结果页

- `财务报表`
  - 页面：`apps/web/src/pages/ReportsPage.tsx`
  - 职责：资产负债表、利润表、现金流量表、快照、差异分析、老板摘要、打印
  - 定位：报表分析页

### 1.4 税务人力

- `税务中心`
  - 页面：`apps/web/src/pages/TaxPage.tsx`
  - 职责：纳税人口径、税率规则、税务事项、申报批次、底稿、个税资料、印花税与附加税、企业所得税准备、打印
  - 定位：税务处理与申报准备页

### 1.5 研发风控

- `研发辅助账`
  - 页面：`apps/web/src/pages/RndPage.tsx`
  - 职责：研发项目、研发成本、工时、趋势、加计扣除资料包
  - 定位：研发财税专题页

- `风险勾稽`
  - 页面：`apps/web/src/pages/RiskPage.tsx`
  - 职责：查看风险发现、对事项触发检查、关闭风险、复盘记录
  - 定位：横向检查与异常闭环页

- `审计日志`

  - 职责：查看审计轨迹、按资源类型/日期过滤
  - 定位：留痕与追溯页

### 1.6 AI 与工具

- `AI 财税助手`

  - 职责：聊天、OCR、上传文件、生成经营事项、保存会话
  - 定位：核心主入口之一

- `制度库`

## 2. 当前真实业务主链

当前系统已经形成的主链是：

1. `AI 财税助手` 或 `经营事项总线`
   - 录入业务、上传图片/PDF、OCR、AI 分析
   - 形成 `business_event`

2. `任务中心`
   - 根据事项生成任务树
   - 分配责任、催办、推进状态

3. `单据中心`
   - 生成并查看业务单据
   - 补齐原始附件、报销票据包、说明资料

4. `凭证中心`
   - 生成凭证
   - 校验、审核、过账

5. `总账中心`
   - 查看分录、余额、日记账
   - 执行期间锁账

6. `财务报表`
   - 汇总成财务三表
   - 形成快照、差异分析、老板摘要

7. `税务中心`
   - 形成税务事项、申报批次、底稿和留档

8. `风险勾稽` / `审计日志`
   - 风险检查、关闭复盘、日志追踪

9. `PDF 导出`
   - 导出部分最终成果

最成熟的闭环：`事项 -> 任务 -> 单据 -> 凭证 -> 总账 -> 报表/税务`

## 3. 页面之间的业务逻辑关系

### 3.1 强关系页面

- `AI 财税助手` -> `经营事项总线`
  - 助手会直接创建事项并可触发分析

- `经营事项总线` -> `任务中心`
  - 事项分析完成后生成任务

- `经营事项总线` -> `单据中心`
  - 事项可生成业务单据和附件要求

- `经营事项总线` -> `凭证中心`
  - 事项可生成凭证草稿

- `经营事项总线` -> `税务中心`
  - 事项可生成税务事项和申报候选

- `单据中心` -> `凭证中心`
  - 单据补齐后支撑凭证审核和过账

- `凭证中心` -> `总账中心`
  - 凭证过账进入总账

- `总账中心` -> `财务报表`
  - 总账分录和余额生成财务报表

- `事项/凭证/单据` -> `风险勾稽`
  - 风险规则以事项、税务、凭证和执行任务为输入

### 3.2 中等关系页面

- `税务中心` 与 `财务报表`
  - 报表结果可作为税务准备参考
  - 税务批次又反向影响留档和导出

- `研发辅助账` 与 `税务中心`
  - 研发加计扣除资料和税收口径有关联

- `研发辅助账` 与 `风险勾稽`
  - 研发归集不足会触发风险

- `审计日志` 与所有页面
  - 作为统一留痕页存在

### 3.3 弱关系页面

- `合同管理`
  - 当前主要是合同台账，与事项、任务、税务、开票、回款链条连接偏弱

- `工资管理`
  - 当前主要是工资计算台账，与事项、任务、凭证、税务联动不强

- `PDF 导出`
  - 当前只覆盖部分导出对象，与系统全链成果覆盖不一致

## 4. 主要问题

### 4.0 当前整改进度

- 已完成：`合同管理 -> 经营事项主线` 第一轮整改
  - 合同页可显式生成关联经营事项
  - 可从合同详情穿透到 `经营事项总线 / 任务中心 / 税务中心 / 凭证中心`
  - `business_events.contract_id` 已在前后端创建链路中正式落字段
- 已完成：`工资管理 -> 事项/凭证/税务主线` 第一轮整改
  - 工资期间可显式生成 `payroll` 事项并自动执行分析
  - 可从工资页直接穿透到 `经营事项总线 / 任务中心 / 税务中心 / 凭证中心`
- 已完成：`PDF 导出 -> 全量成果导出入口` 第一轮收敛
  - 已统一汇总工资、报表、税务底稿、资料包、凭证的导出入口
  - `财务报表` 页已回归“查看/快照/分析”定位，并指向统一导出中心

### 4.1 主链外孤岛

#### 问题 1：合同管理原先没有进入经营事项主线

- 当前 `合同管理` 能建合同、关停合同
- 但没有自动或半自动生成：
  - 经营事项
  - 执行任务
  - 单据要求
  - 税务事项
  - 回款/开票链路

影响：
- 合同页成为独立台账
- 无法自然进入收入确认、回款跟踪、税务检查

当前状态更新：
- 以上问题已完成第一轮修正，合同页不再是完全孤立台账
- 但仍未完成：
  - 合同自动触发任务链
  - 合同自动触发开票/回款/收入确认链
  - 合同与税务事项、单据、凭证的更强关系模型

#### 问题 2：工资管理原先没有进入事项/凭证/税务主线

- 当前 `工资管理` 能算工资、确认工资
- 但缺少明确的：
  - 工资事项生成
  - 工资凭证生成
  - 个税/社保/公积金穿透入口
  - 风险页回跳

影响：
- 工资业务与税务中心之间靠操作者自己理解，不够流程化

当前状态更新：
- 以上问题已完成第一轮修正，工资页不再只是独立工资台账
- 但仍未完成：
  - 工资事项与工资记录的更强关系模型
  - 工资事项自动回写到个税/社保/公积金复核台账
  - 工资事项与风险页的专门回跳入口

### 4.2 页面职责重叠

#### 问题 3：财务报表 与 PDF 导出职责重叠

- `财务报表` 已经能：
  - 打印
  - 保存快照
  - 差异分析
  - 资料包导出

- `PDF 导出` 又单独做：
  - 报表导出
  - 凭证导出
  - 工资导出

影响：
- 两处都像“最终出口”
- 用户不容易判断报表类导出该去哪一页

当前状态更新：
- 已完成第一轮收敛：
  - `财务报表` 主要负责查看、快照、差异分析、老板摘要
  - `PDF 导出` 主要负责最终导出与打印版入口
- 仍待后续深化：
  - 单据中心正式单据导出接入
  - 研发资料包的可打印版接入
  - 更完整的批量导出能力

### 4.3 导出覆盖不足

#### 问题 4：PDF 导出中心的覆盖范围与名称不匹配

当前只支持：
- 工资汇总 / 工资条
- 报表快照
- 凭证

缺少：
- 单据中心正式单据
- 税务底稿
- 风险复盘记录
- 研发资料包
- 月结 / 审计 / 稽核资料包

当前状态更新：
- 已补入：
  - 税务底稿
  - 企业所得税准备稿
  - 月结 / 审计 / 稽核资料包
- 仍缺：
  - 单据中心正式单据
  - 风险复盘记录
  - 研发资料包打印版

### 4.4 导航顺序不够贴近真实使用流程

#### 问题 5：AI 财税助手位置偏后

真实业务中，用户最常见的起点应是：
- `AI 财税助手`
- `经营事项总线`

但当前侧栏把 `AI 财税助手` 放在较后位置，更像辅助工具。

影响：
- 导航表达的“主入口”与系统真实工作流不一致

### 4.5 残留页面/路由歧义

#### 问题 6：BossQAPage 仍存在但不在导航主流里

- `App.tsx` 中存在 `BossQAPage`
- 但 `boss-qa` 路由直接跳转到 `/assistant`
- 导航中也没有独立入口

影响：
- 残留页面会增加维护歧义

## 5. 主要缺失

### 5.1 缺少合同流闭环

建议补齐：
- 合同 -> 事项（已完成第一轮）
- 合同 -> 任务
- 合同 -> 开票 / 回款 / 收入确认
- 合同 -> 税务事项

### 5.2 缺少工资流闭环

建议补齐：
- 工资计算 -> 工资事项（已完成第一轮）
- 工资事项 -> 凭证草稿
- 工资事项 -> 个税 / 社保 / 公积金资料
- 工资事项 -> 风险勾稽

### 5.3 缺少统一最终导出目录

建议把 `PDF 导出` 做成真正的总出口，至少覆盖：
- 单据
- 凭证
- 报表
- 税务底稿
- 工资资料
- 风险复盘
- 研发资料包
- 月结 / 审计 / 稽核资料包

当前状态更新：
- 该项已完成第一轮整改，但仍未覆盖单据正式模板、风险复盘和研发资料包打印版

### 5.4 缺少更多双向穿透

建议加强：
- `工资管理` -> `税务中心`
- `合同管理` -> `经营事项总线`
- `风险勾稽` -> `合同管理` / `工资管理`
- `审计日志` -> 具体对象详情

## 6. 整改优先级

### P0

1. 深化 `合同管理` 到履约/开票/回款/收入确认主线
2. 深化 `工资管理` 到个税/社保/公积金/风险闭环
3. 扩大 `PDF 导出` 到单据、风险复盘、研发资料包

### P1

4. 扩大 `PDF 导出` 的对象覆盖范围
5. 增强 `工资管理`、`合同管理` 与 `税务中心`、`风险勾稽` 的穿透
6. 调整导航，把 `AI 财税助手` 和 `经营事项总线` 提前

### P2

7. 清理 `BossQAPage` 这类残留入口
8. 补更多页面级业务说明和流程图跳转

---

## 7. 推荐下一步

最合理的下一步不是继续加零散页面，而是优先做：

1. `合同管理从事项入口继续深化到履约链`
2. `工资管理从事项入口继续深化到税务与风险闭环`
3. `PDF 导出中心继续扩大到单据 / 风险 / 研发资料`

其中第 1 步优先级最高，因为合同页虽然已接入事项主线，但还没有形成收入、回款、开票、税务的一体化履约闭环。
