## 二、业务链路升级（Phase 9 · ✅ 已完成）

### 背景问题
经营事项总线、单据中心、财务报表、税务中心、研发辅助、风险勾稽、审计日志等页面**信息密度高、内容繁多**，页面内与页面间的**业务先后顺序与逻辑关系**对使用者不够清晰。

### 升级目标（六项可用性原则）
1. **操作逻辑简单**：每页聚焦「当前该做什么」，弱化次要信息。
2. **有引导**：显式呈现「上一步 → 当前 → 下一步」在整体财税链路中的位置。
3. **可重复查看**：任何节点可回看历史与依据，不丢上下文。
4. **可过程跟踪**：每个业务对象有状态时间线，跨页可追踪。
5. **可对内对外管理与申报**：内部协同 + 对外申报/归档一键直达。
6. **沉淀为企业财税管理资料**：最终形成可审查、可留档的完整资料包。

### 统一财税业务链路（贯穿全站的主线）
```
经营事项 → 任务分派 → 单据补齐 → 凭证记账 → 账簿过账
        → 财务报表 → 税务申报 → 风险勾稽 → 归档留档 → 审计追溯
```
